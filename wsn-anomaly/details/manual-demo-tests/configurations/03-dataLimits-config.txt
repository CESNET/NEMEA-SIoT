# Below configuration examples are focused on detection of violation of defined data limits. In configuration profiles are all available items with defined soft and hard limits. The aim is to detects unexpected violations agaist defined profilies configuration. The data were generated by virtual sensors in BeeeOn gateway. 
# If you want to apply a configuration for wsn-anomaly detector, uncomment specific line and run wsn-anomaly with this configuration file.
# Detailed description see above each configuration.

#======================================================================================
#***************************************************************************************
#******************************** TEST6: Data Limits ***********************************
#***************************************************************************************
# DESCRIPTION #
# Soft and hard limits for receiving data are specified by the first five attributes in profile item description. The first two attributes represents min and max soft limits. The next two attributes represent min and max hard limits. The last (fiveth) attribute represents the grace period for soft limit.
# With this configurating you can use these recorded traffic files: onExport-dataLimit-virtual.log, onExport-dataLimit-virtual-time.log

# CONFIGURATION #
#VALUE: 8;8;0;simple;-;-;-;profile(moving_average,moving_variance,moving_median,average,);moving_average(4,10,2,12,5,-,-,0,0,);moving_variance(10,18,5,25,5,-,-,0,0,);average(4,10,2,12,5,-,-,0,0,);moving_median(4,10,2,12,5,-,-,0,0,);export(-,);
